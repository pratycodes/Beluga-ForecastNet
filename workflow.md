# Product Requirements Document (PRD)

## Project Title

LFS-HDLBWO: Short-Term Load Forecasting using CBLSTM-AE and Beluga Whale Optimization

## Objective

Develop a short-term electrical load forecasting system that combines deep learning and metaheuristic optimization to accurately predict future electricity consumption from historical load and auxiliary variables.

The system should reproduce the methodology proposed in:

* CBLSTM-AE (Jogunola et al., 2022)
* LFS-HDLBWO (Asiri et al., 2024)

## Business Goal

Improve forecasting accuracy while maintaining computational efficiency suitable for deployment in smart-grid environments.

Target outcomes:

* Minimize forecasting MSE
* Improve generalization across datasets
* Reduce model tuning effort through automated hyperparameter optimization

# Functional Requirements

## FR-1 Data Ingestion

System shall accept:

* Historical load values
* Weather variables
* Calendar variables

Supported variables:

* Load
* Temperature
* Humidity
* Wind Speed
* Dew Point
* Weekday Indicator
* Weekend Indicator
* Holiday Indicator

Output:

Tensor dataset suitable for deep learning.

---

## FR-2 Data Cleaning

System shall:

* Detect missing values
* Replace missing values using moving average smoothing

Output:

Cleaned time-series dataset.

---

## FR-3 Z-Score Normalization

System shall normalize all features using:

z = (x - μ) / σ

Output:

Scaled feature matrix.

---

## FR-4 Rolling Window Generator

System shall transform time-series data into supervised learning samples.

Default configuration:

* Input Window = 14 timesteps
* Forecast Horizon = 1 timestep

Example:

Input:
[t-13 ... t]

Target:
[t+1]

Output:

3D tensor:

(N, Window_Size, Features)

---

## FR-5 CBLSTM-AE Forecasting Model

Architecture:

Input (8 Features)
↓
Conv1D(64)
↓
Conv1D(64)
↓
MaxPooling1D
↓
BiLSTM(128)
↓
Flatten
↓
RepeatVector
↓
LSTM(64)
↓
TimeDistributed(Dense(32))
↓
TimeDistributed(Dense(1))
↓
Forecast

Training configuration:

* Optimizer: Adam
* Learning Rate: 0.001
* Epochs: 70
* Batch Size: 160
* Validation Split: 0.33
* Activation: ReLU

Output:

Predicted load values.

---

## FR-6 Beluga Whale Optimization

Purpose:

Automatically tune model hyperparameters.

Optimization Variables:

* Conv Filters
* BiLSTM Units
* Decoder LSTM Units
* Learning Rate
* Batch Size

Objective Function:

Minimize Validation MSE

Fitness:

MSE = Mean((Prediction - Target)^2)

---

## FR-7 Model Selection

System shall:

* Train candidate models generated by BWO
* Evaluate validation MSE
* Store best-performing configuration

Output:

Best Hyperparameter Set

---

## FR-8 Final Training

System shall:

* Rebuild CBLSTM-AE using optimal parameters
* Train on full training dataset

Output:

Final forecasting model.

Visualizations:

# Success Criteria

The project is successful if:

* BWO discovers a configuration outperforming the default CBLSTM-AE.
* Forecasting error is lower than baseline LSTM models.
* The model generalizes to unseen validation data.
* End-to-end training completes successfully and produces reproducible forecasts.
